Сервис для хранения медицинских данных. Будет простой веб-интерфейс, с помощью которого можно загружать фотографию и редактировать данные в профиле.
По фотографию высчитывается уникальная строка, возвращаемая как закрытый ключ. Уязвимости предлагаю такие:
1. Вместе с фотографией отсылается ID запроса, причём ID не случайный, а допустим порядковый номер, и закрытый ключ возвращается в соответствии с этим ID.
2. Не уникальность закрытого ключа. Закрытый ключ зависит не от содержимого фотографии, а от её длины в байтах. (например)
3. Уязвимость в структуре хранения данных. Все данные пользователя будут храниться в обычном текстовом файле и перед каждой записью будет стоять её длина. Можно добавить функцию изменения данных, которая не будет пересчитывать длину. Тогда можно будет читать стоящую после запись.


1. Веб интерфейс - JavaScript
2. Серверную часть на Python(можно какой-нибудь фрэёмворк типа торнадо, можно тупо "Многопоточный асинхронный TCP сервер")
3. Функции записи данных в БД, извлечения, и т.п.
4. Придумать алгоритм генерации строки по байтам картинке
